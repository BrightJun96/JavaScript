WEB API는 비동기 처리 방식으로 작동하나?

WEB API : setTimeout , setInterval , HTTP요청 , 이벤트 핸들러 ,DOM API


WEB API는 브라우저 객체이므로 이벤트 루프가 지원이 된다.
때문에 비동기적으로 작동할 수 있다?

즉, 비동기적으로 작동할 수 있는 것은 브라우저에 내장되있는 이벤트 루프 기능 때문이다?

이벤트 루프는 동시성을 지원한다.(멀티스레드)
즉 , 브라우저는 멀티 스레드로 동작한다.

비동기 함수의 콜백함수는 모든 실행컨텍스트가 종료된 이후에 태스크 큐에서 꺼내져 실행된다.

때문에 모든 콜백함수가 비동기적으로 작용하는 것이 아니라
WEB API와 관련된 함수들이 비동기적으로 작용하는 것이다.

비동기 함수에서 값을 처리하기 위해서는
비동기 함수내에서 처리해야한다.
그 방법으로서는 함수의 콜백함수에서  값을 처리하는 것이다.
